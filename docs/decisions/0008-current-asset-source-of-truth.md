# 0008: 当前 MVP 资产数据唯一事实来源

日期：2026-07-12

## 状态

Draft

## 背景

当前根目录原生 JavaScript 静态入口是种子用户试用版本的主体验候选入口。该入口中的资产录入、修改、持仓状态和恢复均以浏览器本地保存的数据为准。

仓库同时存在 Database、API、React 入口以及面向未来的账户、交易、持仓和私人资产表结构。为避免把未来能力、实验性实现或公共行情存储误认为当前 MVP 已启用的私人资产云同步，需要明确当前阶段的唯一事实来源和 Database 边界。

## 决策

当前 MVP 的资产数据唯一事实来源（Source of Truth）为：

- 浏览器 `localStorage` 中保存的 asset snapshot。

Database 当前仅用于：

- 公共行情数据；
- 未来资产云同步、持久化和其他服务端能力的探索。

Database 当前不是 MVP 私人资产数据的事实来源。

## 边界

- 根目录原生 JavaScript 静态入口是当前种子试用主入口。
- 当前资产录入、修改、持仓状态和本地恢复均以 `localStorage` asset snapshot 为准。
- 公共行情、基金净值、汇率和证券基础信息可以由 API 获取并在 Database 缓存。
- Database 中已有或实验性的私人资产表、API 和模型，不代表当前 MVP 已启用资产云同步。
- React 入口中的 Demo 数据不是当前 MVP 的资产事实来源。

## 原因

- 与当前根目录静态 MVP 的真实数据流一致。
- 与当前阶段私人资产本地保存为主、尚未启用完整云同步的产品边界一致。
- 避免多个入口、API 内存状态或实验性数据库表被误认为并行事实来源。
- 为后续资产模型和云同步迁移提供明确的起点与回滚基线。

## 备选方案

### Database 作为当前私人资产事实来源

当前不采用。现阶段尚未完成真实用户身份映射、服务端鉴权、稳定 schema、同步冲突处理、迁移、恢复和删除闭环，不具备替代本地事实来源的条件。

### localStorage 与 Database 双事实来源

不采用。双事实来源会导致资产数量、成本、价格、持仓状态和更新时间发生分叉，且当前没有可用的冲突解决规则。

## 后果

- 当前 MVP 的私人资产读写和恢复逻辑继续以本地 asset snapshot 为准。
- Database 中的私人资产相关结构继续视为未来能力探索，不得作为已上线能力对外承诺。
- 公共行情数据仍可通过 API 和 Database 获取、缓存和更新，但不因此改变私人资产的事实来源。
- 后续迁移到交易账本或云端资产事实来源时，必须另行评审数据、隐私、兼容、迁移和回滚影响。

## 重新评估条件

满足以下条件时，应重新评估并由新的 Decision Log 替代或废弃本决策：

- 产品负责人批准启用私人资产云同步；
- API 已使用真实用户身份并完成服务端鉴权和数据隔离；
- 私人资产持久化 schema、版本、校验、迁移和恢复路径稳定；
- 已定义本地与云端的同步、冲突、删除和回滚规则；
- React 或其他入口准备替代根目录静态 MVP 成为唯一正式主入口。
