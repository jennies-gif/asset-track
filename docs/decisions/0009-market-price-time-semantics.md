# 0009: 公共行情价格时间语义

日期：2026-07-13

## 状态

Accepted

## 背景

此前股票、数字资产和贵金属共用 `tradeDate + closePrice` 结构，部分最新报价会把请求日期写成交易日，导致周末股票出现不存在的收盘日期，也使数字资产即时价混入日频历史。

## 决策

- 股票和场内 ETF 当前估值与历史均使用数据源返回的真实交易日收盘价。
- 场外基金使用基金净值日期与单位净值。
- 数字资产当前估值使用用户检查价格时取得的 Binance ticker 最新参考价；历史与日频计算只使用已经结束的 Binance UTC 日 K 收盘价。
- 贵金属 latest 接口价格标记为参考价，不表达为交易所收盘价。
- `sourceFetchedAt` 只表示抓取时间，不代替价格日期或价格时点。
- 读取旧缓存时按照资产类型和来源推断价格类型；股票 quote 行不得参与收盘价选择。

## 当前存储

当前 JSON 分片与 PostgreSQL `raw_payload` 保存：

- `priceKind`：`close`、`unit_nav`、`latest`、`reference` 或 `snapshot`；
- `priceAt`：带时区的价格时点；
- `marketTimezone`：市场时间口径；
- `sourceTimestamp`：来源声明的时间；
- `sourceFetchedAt`：系统抓取时间。

PostgreSQL runtime 表暂不新增结构化列，避免在未完成生产备份、迁移和回滚验证前自动执行 DDL。需要依赖这些字段的读取以 `raw_payload` 为准。这是当前种子版的过渡限制。

## 后果

- 周末检查股票价格时继续显示最近真实交易日收盘。
- 数字资产主表可随检查更新最新价，但历史曲线不会被即时价污染。
- 当前组合允许混合估值时点，界面必须显示各资产价格口径和时间。
- 旧的周末 quote 缓存可继续保留，但不会参与股票估值；生产清理需要单独审批。

## 后续迁移

完成生产行情表备份与回滚演练后，再评估把价格类型和价格时点迁移为 PostgreSQL 结构化列。迁移前后必须验证 JSON fallback 与 PostgreSQL 返回一致。
