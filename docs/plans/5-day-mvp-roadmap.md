# 2026-07-10 种子用户试用前 Roadmap

目标日期：2026-07-10（周五）  
当前日期：2026-07-05（周日）  
版本定位：公开 beta / 种子用户试用版，而不是正式商业版。

目标：上线一个可信、能录入、能看懂、能备份、能复盘的本地优先 MVP，让第一批真实用户完成核心路径并给出高质量反馈。本轮不追求完整 To C 架构，不做自动投顾、服务端云同步、复杂社区、全量行情、订阅支付或大规模 React 重构。

## 当前产品判断

- 当前主体验是根目录静态 MVP：`index.html`、`src/`、`public/` 和 `dist-static` 构建产物。
- MVP 首页应直接进入资产概览或核心工作流，不优先做产品营销型 Landing Page。
- 用户资产、交易、复盘和设置仍默认保存在浏览器 `localStorage`，不是云端资产账户。
- Supabase Auth 可作为账号入口，但当前资产和复盘尚未云同步；登录后也不能承诺跨设备恢复资产数据。
- 行情 API 和本地行情缓存是增强能力，不是本轮上线的核心承诺；价格可能延迟、缺失或需要用户手动补录。
- 当前资产主库已生成约 1 万条记录，但 A 股和港股覆盖仍有限，不应宣传全市场覆盖。
- 社区/观点广场不进入本轮主体验，不做评论、点赞、排行、跟单或公开资产展示。

## 上线判定

- [ ] 种子用户可以通过 HTTPS 访问 beta 版本。
- [ ] 用户能在 10 分钟内完成：进入产品、添加资产、记录一笔交易、查看总览、查看分析、写一条复盘、导出 JSON 备份、发送反馈。
- [ ] 登录入口、行情同步、资产识别、数据备份和风险提示的能力边界清楚，不制造“已云同步 / 实时行情 / 全市场覆盖”的误解。
- [ ] 价格缺失、手动价格、成本价暂估、估算趋势、导入覆盖和本地数据丢失风险都有明确提示。
- [ ] 页面不出现收益承诺、保本暗示、跟单引导、个性化投资建议、真实密钥、真实账户或真实隐私样例。
- [ ] `npm test`、`npm run typecheck`、`npm run build:static` 和 `npm run build` 通过。
- [ ] 至少完成 3-5 名种子用户邀约，并准备试用说明、反馈入口和反馈整理表。
- [ ] 完成桌面和移动端截图级或人工浏览器验收，确认无白屏、遮挡、不可点击、文本严重溢出或弹窗无法操作。

## P0：上线前必须完成

- [ ] 将对外表述统一为“公开 beta / 种子用户试用版”，避免“正式商业版”预期。
- [ ] 收敛登录入口：隐藏或明确标注 Google、Apple、密码重置等尚未接入能力；邮箱登录也必须说明资产数据仍保存在本机浏览器。
- [ ] 强化本地数据边界：登录弹窗、总览数据状态、备份入口、试用说明中都要一致表达“不会主动上传资产数据，需导出 JSON 备份”。
- [ ] 强化行情边界：同步价格是增强能力，失败不影响手动录入和本地计算；不承诺实时、完整或全市场覆盖。
- [ ] 强化资产识别边界：支持手动录入和部分资产自动识别，不宣传全量 A 股、港股或基金覆盖。

- [ ] 压测线上首屏和资产搜索体验，重点观察 `instrumentRegistry.generated.js` 体积对加载和输入响应的影响。
