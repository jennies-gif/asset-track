# 本周五种子用户上线 Todo

目标日期：2026-06-05（周五）

目标：上线一个可信、能录入、能看懂、能备份、能复盘的本地优先 MVP，拿到第一批种子用户真实反馈。本周不追求完整 To C 架构，不做自动投顾、服务端云同步、复杂社区、全量行情或大规模 React 重构。

## 上线判定

- [ ] 种子用户可以通过 HTTPS 访问产品。
- [ ] 用户能在 10 分钟内完成本地登录、录入资产、查看总览、查看分析、写一条复盘笔记和导出备份。
- [ ] 价格缺失、手动补录、导入失败和本地数据风险都有明确提示。
- [ ] 页面不出现收益承诺、跟单暗示、个性化投资建议或真实隐私样例。
- [ ] `npm test`、`npm run typecheck`、`npm run build` 通过。
- [ ] 至少完成 3 名种子用户的试用邀约，并准备反馈收集表。

- 服务端注册登录、云同步和多设备同步不进入周五版本；当前只保留本地用户和浏览器本地存储。

## 周四 2026-06-04：全流程验收和部署准备

手动验收路径：

- [ ] 录入一笔股票或 ETF，不填写当前价，确认持仓显示待获取价格且估值暂按成本价。
- [ ] 录入一笔现金资产，确认表单显示现金金额，市场和账户类型自动切换。
- [ ] 录入一笔数字资产或黄金类资产，确认分类、市场和币种符合预期。
- [ ] 在总览查看总资产、区间收益、资产类别分布、市场分布和账户列表。
- [ ] 在分析中查看区间变化拆解、贡献排行，并手动补录一次价格或净值。
- [ ] 对一笔资产做分批卖出，再做清仓，确认历史持仓和已实现收益展示正常。
- [ ] 点击清仓复盘弹窗中的“去写复盘笔记”，确认能进入对应模板并关联资产。
- [ ] 导出 JSON 备份，刷新页面后粘贴导入，确认能恢复数据。
- [ ] 粘贴错误 JSON，确认导入失败能指出原因。

部署准备：

- [ ] README 更新周五上线说明、安装、启动、构建、测试、部署和已知限制。
- [ ] 准备静态部署到 HTTPS 环境。
- [x] 准备上线冒烟测试清单和回滚方案。
- [x] 准备种子用户邀请文案、试用说明和反馈表。
- [ ] 暂不实现产品内 onboarding；先在试用说明中列出核心路径，后续再做总览页试用清单。

验收：

- [ ] 跑通 `npm test`、`npm run typecheck`、`npm run build`。
