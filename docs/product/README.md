## 维护规则

- MVP 范围以 `mvp-prd.md` 为主要基线，其他文档通过链接引用，不复制正文。
- 稳定事实、当前阶段、历史计划和阶段性审计应明确区分。
- 重大决定进入 Decision Log；用户可感知变化进入产品 Changelog。
- 文档结论、状态、负责人或日期发生变化时，应同步更新索引和最后更新时间。
- 未经产品负责人确认的假设应明确标记，不得写成已确认事实。
