# 分析模块产品调研与 MVP 建议

本文面向「资产轨迹」分析 tab 的产品打磨，目标用户是中文长期理性投资者。分析模块不应做成交易信号页，也不应输出个性化买卖建议；它应帮助用户理解：当前资产结构是否清晰、组合变化来自哪里、风险和集中度是否可见、数据是否可信。复盘模块可以接收分析页的可选跳转，但两者不应强绑定。

## 0. 总览 tab 与分析 tab 的边界

当前总览 tab 已经承担资产工作台职责，包含总资产、收益率、总资产/收益率变化曲线、资产类别分布、市场分布、账户分布和分布详情。分析 tab 不能重复做一套更复杂的总览，否则用户会分不清“看现在去哪、看原因去哪”。

两个 tab 应按用户问题区分：

| Tab | 用户进来时的问题 | 产品定位 | 展示重心 | 不应承担 |
| --- | --- | --- | --- | --- |
| 总览 | 我现在整体是什么状态？钱在哪里？最近大概变多还是变少？ | 资产仪表盘 / 每日工作台 | 当前规模、区间趋势、资产类别/市场/账户分布、可快速定位的 Top 持仓 | 不解释复杂原因，不做估值判断，不给操作清单 |
| 分析 | 为什么变了？风险在哪里？哪些数据需要补齐？组合是否过度集中？ | 资产解释与数据体检 | 变化原因拆解、贡献排行、风险灯、数据可信度、暴露分析 | 不重复大面积分布图，不做行情终端，不承担复盘主流程 |

### 总览 tab 应坚持“看得见现状”

总览适合回答：

- 当前总资产是多少，按用户选择的计价货币统计。
- 最近 1 日、1 月、近 3 月、今年或自定义区间，大方向是上升还是下降。
- 资产主要分布在哪些类别、市场和账户。
- 点开某个类别或市场后，里面有哪些主要持仓。
- 哪个账户或市场占比最大，是否需要去资产台账查看明细。

推荐保留和强化的总览内容：

- 顶部指标：总资产、区间收益率；如果增加指标，应优先加入“未实现收益”和“数据状态摘要”，不加入复杂变化原因拆解。
- 趋势图：只展示总资产或收益率趋势，服务“整体走势”。
- 分布区：资产类别、市场、账户分布，服务“钱在哪里”。
- 分布详情：Top 持仓、市值、未实现收益，服务“快速定位”。
- 空状态和入口：引导去资产页录入、去分析页查看原因。

总览不建议出现：

- PE/PB、估值分位、财务指标。
- 净投入、价格、汇率、分红、费用的详细拆解。
- “需要加仓/减仓/清仓”之类操作暗示。
- 大量新闻事件或社区观点。

### 分析 tab 应坚持“讲清楚原因”

分析适合回答：

- 本区间组合变化由哪些因素贡献。
- 贡献最大和拖累最大的资产分别是谁。
- 当前组合有哪些风险已经超过阈值，例如集中度、回撤、市场/币种暴露、数据缺失。
- 哪些资产因为贡献异常、价格过期、缺来源或数据不一致，需要进入数据维护。
- 某个资产当前估值处在什么位置，但只表达“历史分位/同类位置/数据不足”，不表达“应该买卖”。

分析可以复用总览的日期区间，但不能复用总览的信息结构。总览使用“资产视角”的分布和趋势，分析使用“原因视角”的拆解和清单。

推荐的入口关系：

- 总览趋势图旁显示轻量入口：“查看本区间变化原因”，跳到分析页的变化原因拆解。
- 总览分布详情显示轻量入口：“查看该类别/市场贡献”，跳到分析贡献排行并带过滤条件。
- 分析中的数据问题跳到资产页补字段或价格。
- 分析页可以提供“关联到复盘”的轻入口，但不把写复盘作为完成分析的必要步骤。

一句话原则：

> 总览回答“我现在在哪里”，分析回答“我为什么到这里，以及下一步该检查什么”。

## 1. 资产管理类竞品分析

本轮竞品应以“个人资产管理、投资组合追踪、复式记账、财富记录”产品为主，而不是行情/社区/交易平台。更贴近的参照物是 Ghostfolio、Beavern、Wealthfolio，以及 Beancount / Portfolio Performance 这类强调数据可解释和可复算的工具。

### 1.1 Ghostfolio

常见内容：投资组合概览、持仓、资产配置、交易记录、基准对比、历史表现、净值/收益、分红、费用、FIRE/财富目标、数据导入导出、自托管和隐私控制。

分析什么：

- 我的总财富、组合收益、不同账户和资产类别的占比。
- 组合表现是否跑赢参考基准。
- 资产配置是否偏离目标，风险是否过度集中。
- 现金流、分红、费用和交易对长期结果的影响。

怎么呈现：

- 首页先给净资产和组合表现，再进入持仓、配置、交易、目标等模块。
- 图表服务于组合追踪：价值曲线、资产配置、账户/类别占比、历史收益。
- 强调自托管、隐私、开源和可导入数据，适合长期记录。

优点：

- 模块边界清楚：Portfolio 记录事实，Analysis/Insights 解释组合。
- 适合长期用户，不以短线行情和新闻流为核心。
- 基准、分红、费用、配置偏离这些能力很适合资产管理场景。

不足：

- 对中文用户的账户类型、本土市场和多币种资产支持需要本地化。
- 如果只复制其指标，会变成“看板型资产管理”，缺少资产轨迹强调的数据可信和变化解释。

值得借鉴：

- 分析页应围绕“组合表现、资产配置、基准、费用/分红、集中度”组织。
- 数据导入导出和隐私说明应是分析可信度的一部分。
- 基准对比可以作为二期，但 MVP 先用现金/宽基基准即可。

### 1.2 Beavern

常见内容：总资产汇总、资产分类、资产分析、Stage Comparison（阶段对比）、Asset Filter（资产筛选）、快照、净值、投资历史和会计式数据组织。其帮助文档把 Analysis 明确拆成 Real-time Analysis 和 Stage Comparison，并支持通过 Asset Filter 按分类、账户、标签等维度筛选资产。

分析什么：

- 当前资产结构是什么样：资产类别、账户、现金、证券、其他资产。
- 某两个阶段之间资产如何变化：总额变化、分类变化、具体资产变化。
- 用户筛选某一类资产后，这一组资产在当前和历史阶段的表现。

怎么呈现：

- Real-time Analysis 更像“当前资产结构体检”。
- Stage Comparison 更像“两个时间点的资产变化对账”。
- Asset Filter 让分析结果围绕用户选择的一组资产重算。

优点：

- 分析模块边界非常清楚：不是行情分析，而是资产事实分析。
- “当前分析”和“阶段对比”两个入口对普通用户很容易理解。
- 筛选器天然适合多账户、多资产、多币种场景。

不足：

- 如果缺少原因解释，用户仍然只能知道“变了多少”，不一定知道“为什么变”。
- 对数据缺口、未解释差异和来源追踪的引导还可以更强。

值得借鉴：

- 分析 tab 首屏应拆成两个清晰问题：“现在结构健康吗？”和“这段时间为什么变了？”
- 日期区间应表达为“阶段对比”，比“区间归因”更直觉。
- 所有分析都应支持按账户、资产类别、市场、币种、标签过滤。

### 1.3 Wealthfolio

常见内容：本地优先的投资组合追踪、持仓、收益、资产配置、交易、分红、账户、图表和桌面端隐私体验。

分析什么：

- 当前组合价值、收益、资产分布和持仓明细。
- 分红、交易和长期收益表现。
- 多账户组合在本地隐私环境下的整体状态。

怎么呈现：

- 桌面工具式界面，强调简洁、私密、可控。
- 以组合和持仓为中心，图表通常围绕价值、收益、分布和现金流。
- 不把社区、资讯和交易信号作为核心体验。

优点：

- 产品气质接近资产轨迹：长期记录、本地优先、隐私友好。
- 信息层级不应过度金融终端化，用户要能快速看懂自己的组合。
- 分红、交易、账户、持仓是长期投资者真正会反复查看的对象。

不足：

- 若分析能力只停留在总额和分布，和总览 tab 容易重叠。
- 需要更明确地引导用户做数据修复和原因解释。

值得借鉴：

- 分析页视觉应保持工具型、紧凑、清晰，不做资讯流。
- 本地数据、导入导出、隐私和数据来源应被显式呈现。
- 分红/费用/现金流应进入长期收益解释。

### 1.4 Beancount / Fava

常见内容：复式记账、账户树、资产负债表、利润表、交易流水、投资持仓、成本、价格、查询、报表和可审计账本。

分析什么：

- 每一笔资产变化是否能回溯到账本交易。
- 资产、负债、收入、费用和投资收益如何在账户体系里流动。
- 某个时间点或时间段的余额、现金流、成本和收益。

怎么呈现：

- 账户树、交易流水和报表是核心。
- 分析来自可查询、可复算的账本，而不是黑箱指标。
- 用户可以从报表下钻到交易明细。

优点：

- 非常符合资产轨迹的数据准确性和审计可追踪原则。
- “先事实，后分析”的设计很稳。
- 对费用、税费、分红、现金流和未解释差异特别有启发。

不足：

- 普通投资者学习成本高。
- 界面更偏账本/技术用户，不适合直接复制。

值得借鉴：

- 分析模块所有数字都应该能下钻到交易、价格和汇率来源。
- 未解释差异必须作为一等公民展示。
- 数据质量、导入批次、手动调整和来源说明是分析可信度的一部分。

### 1.5 Portfolio Performance

常见内容：净值表现、收益率、现金流、资产配置、分类法、证券账户、交易、分红、费用、税费、基准、风险指标和报表。

分析什么：

- 投资组合真实收益，而不是只看市值变化。
- 现金流、费用、税费、分红和价格变化对结果的影响。
- 资产配置、分类、账户和证券对组合的贡献。
- 组合与基准的长期表现差异。

怎么呈现：

- 左侧功能树 + 中间报表/图表，适合深度分析。
- 支持多种分类法和报表视角。
- 强调收益率、现金流和绩效计算。

优点：

- 非常适合学习“收益表现”和“现金流调整后收益”的表达。
- 分类法和基准能力值得二期参考。
- 对费用、税费、分红的严谨处理适合长期投资者。

不足：

- 功能专业且密集，MVP 不能照搬。
- 中文个人用户首屏需要更明确的问题导向。

值得借鉴：

- MVP 应先把“现金流调整后的变化”和“简单基准对比”做清楚。
- 资产分类和账户筛选要为分析服务，而不是只做分布图。
- 风险收益指标应逐步引入，避免首版压垮用户。

## 2. 长期投资者真正关心的问题

长期投资者打开分析 tab，表面上是在看收益、估值和风险，实际是在问以下问题：

1. 当前资产结构是否清晰：
   - 资产分布在哪些账户、市场、类别和币种。
   - 是否存在单一资产、单一市场或单一币种过度集中。
   - 是否有价格、汇率、来源或交易记录缺失。

2. 当前贵不贵：
   - 当前估值处在自身历史什么位置。
   - 相比同类资产是否明显偏贵或偏便宜。
   - 价格上涨是盈利增长驱动，还是估值扩张驱动。

3. 风险能否承受：
   - 最大回撤、波动、集中度、币种暴露、单一市场暴露是否超过承受范围。
   - 某个资产下跌 20%、30%、50% 时，组合会损失多少。
   - 数据缺失是否让风险被低估。

4. 这段时间变化是否能解释清楚：
   - 净投入、价格变化、汇率变化、分红、费用、税费和手动调整分别贡献多少。
   - 各项加总后是否能对齐资产变化。
   - 无法解释的差异是否被明确标记。

5. 数据是否可信：
   - 价格和汇率是否有来源和日期。
   - 手动补录、导入批次、修改记录是否可追踪。
   - 缺失或过期数据是否会影响分析结论。

6. 是否需要进一步查看明细：
   - 哪些资产贡献或拖累最大。
   - 哪些账户、市场、类别或币种需要单独筛选查看。
   - 哪些数据问题需要回到资产台账补齐。

因此，分析模块首屏不应问“今天涨跌多少”，而应回答：

- 这段时间我的组合为什么变了。
- 哪些资产贡献最大、拖累最大。
- 哪些风险或集中度值得关注。
- 哪些资产、账户或市场贡献最大。
- 哪些数据需要补齐或核对。

## 3. 三类内容取舍

### 3.1 可直接借鉴竞品的内容

1. 当前资产结构分析
   - 借鉴对象：Beavern Real-time Analysis、Ghostfolio 概览、Wealthfolio 组合追踪。
   - 原因：用户需要先确认“我现在的资产结构是否健康”，这和总览的区别是分析页要给结构判断和问题提示。
   - 展示：资产体检卡 + 资产类别/市场/账户/币种暴露表 + 集中度提示。

2. 阶段对比
   - 借鉴对象：Beavern Stage Comparison、Beancount/Fava 按日期查询余额。
   - 原因：“区间”对普通用户抽象，“阶段对比”更像资产管理工具里的对账。
   - 展示：起点资产、终点资产、净变化、净投入、投资结果、未解释差异。

3. 变化原因拆解
   - 借鉴对象：Portfolio Performance 的绩效/现金流思路、Beancount 的账本可复算性。
   - 原因：用户不只要知道资产变多还是变少，还要知道来自净投入、价格、汇率、分红、费用还是调整。
   - 展示：变化原因列表 + 可展开明细 + 数据来源。

4. 多维筛选
   - 借鉴对象：Beavern Asset Filter、Portfolio Performance 分类法。
   - 原因：多账户、多市场、多币种用户不会只看全组合，需要按账户、资产类别、市场、币种、标签切片。
   - 展示：顶部筛选条，所有分析卡随筛选重算。

5. 数据来源和质量提示
   - 借鉴对象：Beancount/Fava 的可审计账本、Ghostfolio/Wealthfolio 的导入导出与隐私定位。
   - 原因：本产品强调可复算和审计。
   - 展示：数据可信度、缺失价格、过期价格、手动数据、未解释差异、备份状态。

### 3.2 竞品有但需简化改造的内容

1. 收益率和历史表现
   - 竞品做法：展示时间加权收益、资金加权收益、净值曲线、基准对比。
   - 改造原则：MVP 先不引入过多专业收益率口径，先明确“总资产变化、净投入后结果、未实现收益”。
   - 推荐输出：阶段对比卡 + 简单历史曲线 + 口径说明。

2. 基准对比
   - 竞品做法：和指数、现金、用户自定义基准比较。
   - 改造原则：MVP 不做复杂全市场基准，先支持现金基准和少量宽基基准。
   - 推荐输出：基准对比小卡，显示“仅作参照，不构成投资建议”。

3. 分红、费用、税费
   - 竞品做法：作为交易流水和绩效的一部分进入报表。
   - 改造原则：MVP 先把它们放进变化原因，不单独做复杂报表。
   - 推荐输出：变化原因拆解中的独立行 + 可下钻交易明细。

4. 风险指标
   - 竞品做法：波动率、最大回撤、风险收益比、集中度。
   - 改造原则：MVP 不做复杂金融风险模型，先做用户能理解的集中度、币种/市场暴露、数据缺失。
   - 推荐输出：风险灯 + 触发原因。

5. 估值分析
   - 竞品做法：资产管理工具通常不会把估值做成首屏核心，更多行情平台才会这样做。
   - 改造原则：MVP 暂不放首屏，作为资产详情二期能力。
   - 推荐输出：资产详情里的历史分位/估值位置，不进入组合分析首页。

### 3.3 竞品没有或不足，可做差异化的内容

1. 数据可信度评分
   - 分析什么：价格来源、价格日期、汇率、交易流水、手动调整、未解释差异。
   - 价值：这是资产轨迹的核心可信差异化。
   - 展示：数据可信度徽标 + 问题清单 + 修复入口。

2. 可复算明细下钻
   - 分析什么：每个变化项对应哪些交易、价格、汇率和手动调整。
   - 价值：把分析结果从黑箱图表变成可审计结果。
   - 展示：变化项展开后显示明细来源、日期、金额和数据状态。

3. 资产筛选后的重算
   - 分析什么：用户选择某账户、某市场、某币种或某资产类别后，所有分析重新计算。
   - 价值：比静态分布图更适合多账户资产管理。
   - 展示：筛选条 + 当前筛选摘要 + 可清除筛选。

4. 组合压力测试
   - 分析什么：单个资产、市场、汇率或整体下跌时组合损失。
   - 价值：比抽象波动率更容易理解风险。
   - 展示：压力测试滑杆或情景表，例如“美股 -20%、美元兑人民币 -5%”。

5. 可选复盘关联
   - 分析什么：某个异常变化或资产明细是否已经关联过笔记。
   - 价值：给需要复盘的用户留入口，但不让分析页变成复盘页。
   - 展示：行内“关联笔记”或“查看关联笔记”，默认不作为主任务。

## 4. 推荐分析内容与展示样式

| 内容 | 解决的问题 | 推荐展示样式 | MVP 取舍 |
| --- | --- | --- | --- |
| 当前资产体检 | 现在结构是否健康 | 顶部资产卡 + 风险/数据状态徽标：集中度、币种、市场、价格日期、未解释差异 | 必做 |
| 阶段对比 | 从起点到终点变了多少 | 起点/终点/净变化/净投入/投资结果五项卡片，名称用“阶段对比” | 必做 |
| 变化原因拆解 | 为什么资产变化 | 紧凑列表：净投入、价格、汇率、分红、费用、税费、调整、未解释差异；每项可展开来源 | 必做，已有基础 |
| 多维筛选 | 我只想看某账户/某市场/某类资产 | 筛选条：账户、资产类别、市场、币种、数据状态、标签；所有卡片随筛选重算 | MVP 简版 |
| 资产贡献排行 | 哪些资产拉动/拖累最多 | 表格：资产、贡献金额、贡献率、权重、数据日期、数据状态 | 必做 |
| 组合暴露 | 是否过度集中 | 暴露表：资产类别、市场、账户、币种、单一资产权重 | 必做，和总览复用但加入风险判断 |
| 数据可信度 | 数字是否可信 | 可信度徽标 + 问题清单：缺价格、价格过期、手动补录、缺来源、未解释差异 | 必做 |
| 数据维护清单 | 哪些数据需要核对 | 清单：未解释差异、价格过期、缺来源、汇率缺失、手动调整待确认 | 必做 |
| 基准对比 | 表现是否有参照 | 现金/宽基基准小卡，显示收益差和口径说明 | 二期 |
| 分红费用税费 | 长期收益被哪些现金流影响 | 现金流明细表或趋势图 | 二期 |
| 估值/财务 | 单个资产基本面是否变化 | 资产详情页，不放分析首页 | 二期 |

## 5. 建议的信息架构

分析 tab 应按资产管理工具的心智组织，而不是行情工具或复盘工具的心智。推荐一级结构是「当前体检、阶段对比、变化原因、贡献与暴露、数据维护」：

1. 当前资产体检
   - 顶部资产卡：当前资产、币种切换、阶段变化。
   - 三个状态徽标：结构风险、数据可信度、来源完整度。
   - 这里回答“现在结构是否健康”，不要重复总览的大面积分布图。

2. 阶段对比
   - 日期选择沿用总览，但标题使用“阶段对比”。
   - 展示起点资产、终点资产、净变化、净投入、投资结果、未解释差异。
   - 用户能一眼明白“这段时间到底变了多少，里面有多少是我自己投入的”。

3. 变化原因
   - 拆成净投入/提现、价格变化、汇率变化、分红/利息、费用、税费、手动调整、未解释差异。
   - 每项给一句解释和可展开明细。
   - 所有数字必须能回到交易、价格、汇率和手动录入来源。

4. 贡献与暴露
   - 贡献排行：贡献最大、拖累最大。
   - 暴露表：资产类别、市场、账户、币种、单一资产权重。
   - 与总览区别：总览展示“钱在哪里”，分析展示“哪里变化最大、哪里数据不可信、哪里风险集中”。

5. 多维筛选
   - 支持账户、市场、资产类别、币种、标签、数据状态。
   - 筛选后重新计算当前体检、阶段对比、变化原因和贡献排行。
   - 这是 Beavern/Portfolio Performance 类工具很值得借鉴的核心能力。

6. 数据维护清单
   - 列出需要核对的数据：未解释差异、价格过期、缺来源、汇率缺失、手动调整待确认。
   - 每项提供“补数据”“查看明细”“查看来源”入口。
   - 可选提供“关联笔记”，但不作为默认主按钮。

## 6. MVP 首版最该做的 5 个模块

### 1. 当前资产体检

理由：总览已经回答“资产在哪里”，分析首屏必须回答“结构是否健康、数据是否可信、是否存在明显集中或缺口”。这能和总览形成明显差异。

推荐展示：

- 当前资产大卡。
- 结构风险、数据可信度、来源完整度三个状态徽标。
- 一句话总结只陈述事实，例如“单一资产占比 34%，2 个资产价格过期，未解释差异为 ¥1,200。”
- 常驻提示：“数据仅供记录和分析，不构成投资建议。”

### 2. 阶段对比

理由：“区间归因”难懂，“阶段对比”更贴近资产管理和对账心智。用户先看起点、终点、净变化、净投入和投资结果，再看原因。

推荐展示：

- 起点资产、终点资产、净变化、净投入、投资结果、未解释差异。
- 每个指标带口径说明。
- 与总览使用同一日期区间。

### 3. 变化原因拆解

理由：这是资产轨迹的核心分析能力。长期用户不只关心赚亏多少，更关心来自投入、价格、汇率、分红、费用、税费还是数据问题。

推荐展示：

- 列表优先，瀑布图二期。
- 每项一句解释。
- 可展开到交易、价格和汇率来源。
- 未解释差异常驻。

### 4. 多维筛选 + 贡献排行

理由：资产管理类竞品的核心不是单一全局分析，而是能按账户、市场、资产类别、币种切片。筛选后再看贡献排行，用户才会真正知道问题在哪。

推荐展示：

- 筛选条：账户、市场、资产类别、币种、数据状态。
- 贡献最大/拖累最大表格。
- 行内显示权重、价格日期、数据状态。

### 5. 数据维护清单

理由：Ghostfolio、Wealthfolio、Beavern 更偏资产追踪，资产轨迹的差异化应是“追踪之后知道数据哪里不可信、哪里需要补齐”。分析页最终应让用户知道下一步该核对什么。

推荐展示：

- 清单项按优先级排序：未解释差异、价格过期、缺来源、汇率缺失、手动调整待确认。
- 每项显示触发原因、影响范围和下一步入口。
- 操作文案使用“补数据”“查看明细”“查看来源”，避免“买入/卖出建议”。

## 7. 首版不建议做或谨慎做

- 不做全量技术指标和短线信号。
- 不做“AI 推荐加仓/减仓/清仓”。
- 不做收益排名、跟单暗示和社区热股榜。
- 不把估值、财务、新闻、机构观点放进分析首页。
- 不自造公允价值、星级评级或“低估/高估”结论。
