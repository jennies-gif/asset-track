## 当前产品阶段

上线准备阶段（种子用户试用版本准备）

当前核心产品闭环已基本实现，产品已超过 MVP 构建中期。但当前尚未形成经过冻结和完整验收、可交给目标用户独立试用的稳定版本，也尚无真实种子用户完成核心路径的验证证据，因此尚未进入小规模用户验证阶段。

## 当前周期

可信种子用户试用版本冻结与核心闭环验收周期

## 本阶段唯一目标

冻结并验收一个私人数据边界真实、损坏数据不会静默替换、能让目标用户独立建立首个清晰可信整体资产视图的根目录静态 MVP 种子试用版本。

## 阶段完成标准

### 一、版本基线

- 明确唯一种子试用主入口。
- 冻结对应版本、部署地址和回滚基线。
- 记录已知问题。
- 不得让新旧两个入口同时作为对外试用主体验。

### 二、核心产品闭环

- 目标用户能在不依赖开发者讲解的情况下，从空白状态独立添加至少 2 个资产。
- 能建立至少两个账户，或包含不同市场、币种的初步组合。
- 总览正确展示组合总值、主要持仓和至少两个实际存在的分布维度。
- 关键结果可以由录入数量、价格和汇率复算。
- 用户能够理解整体资产、主要分布和哪些数据仍需补充。

### 三、数据可信与安全

- 同步价格、手动价格、成本价暂估、价格缺失或过期、成本缺失等核心状态均正确展示。
- 缺失或估算数据不得显示为正常同步数据。
- 产品关于本地保存、服务器传输和行情 API 的说明必须与真实代码完全一致。
- JSON 导出、错误文件拦截、导入预览、取消和确认恢复在当前试用版本实测通过。
- 本地数据解析失败时不得静默回退为正常 Demo 数据，必须明确告知用户并提供恢复或处理路径。

### 四、测试准备

- 当前版本在 390px、768px 和 1280px 下完成核心路径验收。
- 不存在白屏、遮挡、主按钮不可点击、表单无法提交、弹窗无法退出或关键内容不可读。
- 种子测试材料以首次组合建立和整体视图可信度为主任务。
- 测试记录必须覆盖完成情况、录入耗时、主要卡点、数据可信理解、与原有方式比较和继续维护意愿。

## 已实现事项

- 已确认产品定位、首批目标用户、第一价值和核心假设。
- 已实现添加资产、账户关联、持仓管理、总览、主要持仓和分布。
- 已实现主要价格与数据状态表达。
- 已实现 JSON 导出、导入校验、预览和确认。
- 已准备种子用户试用、冒烟测试和反馈材料。
- 已有金融计算、价格状态和行情 API 等自动测试。

以上主要表示能力已实现，不代表已经经过真实目标用户验证。

## 正在进行

- 审计私人资产数据的实际网络传输行为，并等待产品负责人确认当前允许的数据边界。
- 修复本地数据损坏时静默回退 Demo 的问题，补充错误告知和恢复路径。
- 收敛当前工作区变更，修复自动检查阻断并形成唯一冻结候选版本。
- 准备与冻结版本对应的 390px、768px 和 1280px 核心路径验收。
- 将种子测试主任务收敛到首次组合建立和整体视图可信度，次级任务不作为首次成功门槛。

当前工作区中的未提交内容不得视为稳定完成能力。

## 当前主要矛盾

已实现能力持续增长，但这些能力尚未收敛成一个数据边界一致、损坏可恢复、可回滚并经过完整验收的唯一种子版本，导致真实用户测试缺少可信且可解释的测试载体。

## 当前有效阻断项

1. **私人资产数据传输行为与产品承诺存在冲突。** 实际代码中的服务器传输行为与产品关于本地保存、数据未上传的说明尚未完全一致。具体允许传输的数据边界仍待产品负责人确认，但代码行为、产品说明和用户知情必须在种子测试前完全一致。
2. **本地数据损坏时可能静默回退为 Demo。** 用户可能把示例数据误认为自己的数据被替换或丢失，当前缺少明确的错误告知和恢复路径，因此不能作为可信资产工具交给目标用户独立试用。
3. **当前没有经过最新完整验收的冻结种子试用版本。** 当前工作区仍有未提交调整，自动检查尚有阻断，核心路径缺少与最新版本对应的桌面和移动端验收证据，部署与回滚基线也尚未确认。

种子测试任务过多不再作为第四个同级阻断项；它在测试准备中通过收敛唯一主任务解决。首次测试只以组合建立和整体视图可信度为主，交易、深度分析、复盘、备份和付费问题作为可选观察，不影响首次核心任务成败。

## 当前主要风险

### 当前阻断风险

- 私人资产上传边界不清。
- 本地数据损坏静默回退 Demo。
- 未冻结版本。
- 当前核心路径缺少与冻结候选版本对应的最新桌面与移动端验收。

### 可以带入种子测试的风险

- 有限行情覆盖和偶发同步失败。
- 依赖用户手动备份。
- 部分趋势是估算。
- UI 尚未达到成熟商业产品水平。
- 当前原生 JavaScript 架构存在维护成本。

以上风险可以带入种子测试的前提是状态表达清楚、主路径可用，且不会损害数据正确性。

### 当前暂不处理

- 完整云同步和多设备一致性。
- 全市场、实时行情和完整公司行为。
- 完整社区。
- 订阅支付。
- 复杂团队权限。
- 大规模 React/TypeScript 主入口替换。
- 完整高级归因与复盘体系。

## 下一阶段进入条件

只有同时满足以下条件，才能进入“小规模用户验证阶段”：

- 唯一试用入口和冻结版本明确。
- 核心路径在桌面和移动端通过验收。
- 私人资产数据边界与产品说明一致。
- 金融计算、可信状态和恢复路径正确。
- 种子测试材料能够单独验证首次录入成本和整体资产视图价值。

进入小规模用户验证阶段后，重点关注：

- 用户能否独立建立初步组合。
- 录入和维护成本是否可接受。
- 整体视图是否比现有方式更清晰。
- 用户是否理解和信任数据状态。
- 用户是否愿意继续维护组合。
