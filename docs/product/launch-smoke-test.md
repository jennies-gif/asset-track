# 上线前冒烟测试和回滚方案

适用范围：根目录静态 MVP 主体验（`index.html`、`src/`、`storage/`）。  
目标：上线前用最短路径确认产品可打开、可录入、可分析、可备份、可反馈，并且没有明显合规或隐私风险。

## 通过标准

- 首页能通过 HTTPS 打开，无白屏、无控制台阻断错误。
- 用户可以在 10 分钟内完成：录入资产、查看总览、查看分析、写复盘、导出备份、发送反馈。
- 价格缺失、估算图表、导入覆盖、本地存储风险都有明确提示。
- 页面不出现收益承诺、保本暗示、跟单引导、个性化投资建议或真实隐私样例。
- `npm test`、`npm run typecheck`、`npm run build` 全部通过。

## 自动检查

上线前在仓库根目录执行：

```bash
npm test
npm run typecheck
npm run build
```

如果本地静态服务已启动，检查核心资源：

```bash
curl -I http://127.0.0.1:4175/
curl -I http://127.0.0.1:4175/src/app.js
curl -I http://127.0.0.1:4175/src/styles/app.css
```

检查敏感和合规文案：

```bash
rg -n "收益承诺|保本|跟单|推荐买|推荐卖|真实账户|身份证|secret|api_key|password|token" . --glob '!node_modules/**' --glob '!apps/web/dist/**' --glob '!package-lock.json'
```

允许出现的结果：

- 文档中的合规禁用词清单。
- 风险提示里的“不构成投资建议”。
- 环境变量示例或行情脚本中的占位密钥字段名。

需要拦截上线的结果：

- 产品页面承诺收益、保本、跟单或推荐具体买卖。
- 真实账户、真实交易记录、真实身份证明或真实密钥。
- 未脱敏的邮箱、手机号、交易所 API Key 或券商凭证。

## 桌面端手动冒烟

建议宽度：1280px。

- 打开首页，确认总览、资产、分析、复盘、设置入口可见且无遮挡。
- 确认页面能看到本地存储、备份、风险提示和数据来源说明。
- 新增一笔股票或 ETF，不填写当前价，确认状态显示待补价格，估值按成本价或明确标注。
- 新增一笔现金资产，确认表单切换到现金金额语义，账户和币种正常保存。
- 新增一笔数字资产或黄金类资产，确认分类、市场、币种和数量正常保存。
- 对一笔持仓记录加仓、减仓，再清仓，确认数量、成本、已实现收益和历史持仓正常。
- 清仓后点击“去写复盘笔记”，确认进入清仓复盘模板并关联资产。
- 打开分析页，确认区间变化拆解、贡献排行、资产分布、风险收益图表都有数据口径标签。
- 导出 JSON 备份，确认文件可以下载。
- 粘贴错误 JSON 导入，确认失败信息明确。
- 粘贴有效 JSON 导入，确认先展示覆盖预览和差异，再由用户确认覆盖。
- 点击页脚“用户反馈”，确认反馈弹窗可打开、可复制模板，邮件草稿收件人为 `jennieshen1027@gmail.com`。

## 移动端手动冒烟

建议宽度：390px 和 768px。

- 打开总览，确认顶部、导航、指标和主操作无重叠。
- 打开资产表单，确认字段单列展示，输入框、选择器、提交按钮可点击。
- 查看持仓表，确认表格可横向滚动，关键列不会被压到不可读。
- 查看分析页，确认图表标题、数据口径、空状态和说明文字不互相遮挡。
- 打开复盘编辑器，确认标题、正文、标签、资产关联和保存按钮可用。
- 打开反馈弹窗，确认在手机宽度下内容可滚动，复制和邮件按钮可点击。

## 上线后 5 分钟检查

- 打开线上首页，确认首屏正常渲染。
- 新增一笔虚构资产，刷新页面后确认本地数据仍保留。
- 导出 JSON，再用错误 JSON 验证导入失败提示。
- 打开反馈入口，确认邮件地址和反馈模板正确。
- 再次确认页面没有真实密钥、真实账户、真实交易记录或收益承诺。

## 回滚方案

静态 MVP 优先使用托管平台的上一个成功部署版本回滚。

1. 保留每次上线前的构建产物、部署版本号或 commit 标识。
2. 上线后若出现白屏、数据丢失、无法保存、无法导出、严重计算错误或严重合规文案，立即停止扩散链接。
3. 在部署平台回滚到上一个通过冒烟测试的版本。
4. 回滚后重新执行“上线后 5 分钟检查”。
5. 将问题记录到 roadmap 或 issue，修复后补充可复现测试，再重新发布。

不建议用临时热修绕过导入导出、持仓计算或风险提示；这些属于信任基础，修复必须可复算、可测试。
