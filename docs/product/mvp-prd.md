# 资产轨迹 MVP PRD

## 目标

资产轨迹 MVP 面向个人投资者，提供资产记录、持仓估值、资产变化归因、复盘笔记、本地或云端备份和基础社区治理能力。

产品只用于记录和分析，不提供自动投顾、交易建议、收益承诺或跟单服务。

## 1.0 产品定位

1.0 优先服务中文跨市场投资者的隐私优先投资复盘与归因需求。产品不以“资产看板”作为唯一价值，而是帮助用户回答三类问题：

- 我的资产事实来源是否完整、可追踪。
- 这段时间资产为什么变化，价格、汇率、现金流和费用分别贡献了多少。
- 当初的投资判断有没有被后续数据验证，哪些经验需要沉淀。

## 1.0 一级模块

1.0 主导航收敛为四个一级模块，顺序为：

- 总览：资产工作台，展示总资产、区间收益、区间收益率、未实现收益、总资产/收益率变化曲线、资产类型分布、市场分布、账户分布和可点击的分布详情；区间变化默认近一年，并允许用户手动筛选开始日期和结束日期。
- 资产：统一承载资产录入、当前持仓、历史持仓和账户，作为所有计算的事实来源。
- 分析：承载组合变化解释、贡献拆解、资产/账户/类别贡献、未归因差异提示、数据维护和备份入口，帮助用户理解资产发生了什么，以及判断数字是否可信。
- 复盘：承载投资复盘笔记、待复盘事项、决策假设和轻量公开复盘入口。

资产和分析的边界：

- 分析回答“资产表现怎么样、为什么变化、哪里值得复盘”。
- 分析中的数据维护回答“数字从哪里来、是否可靠、是否缺失、如何导入导出和备份”。

社区不作为 1.0 独立一级模块，先以内嵌在复盘模块中的“公开复盘/观点广场”承载，避免产品早期偏离工具主线并降低合规和冷启动压力。

## 首版用户

- 个人投资者：记录多账户、多币种、多资产组合。
- 轻量团队用户：共同维护资产记录和复盘资料，但首版不做复杂组织权限。
- 长期复盘用户：希望把交易原因、价格变化和后续验证沉淀为可追踪笔记。

## MVP 必须完成

- 用户注册、登录和会话管理。
- 账户、资产、交易和持仓录入。
- 资产录入默认使用快速模式，优先收集用户天然知道的最少必填字段：资产名称、分类、市场、账户类型、账户名称、币种、持有数量/份额和平均成本价；账户类型是上层大类，默认提供证券账户、基金账户、现金账户、数字资产账户、养老金/长期账户、模拟/观察账户和其他，并允许用户自定义账户类型；账户名称与账户类型联动，按类型提供常见平台或账户选项，例如数字资产账户下提供 OKX、Binance、Bybit 和冷钱包，同时允许用户新增自定义账户名称；市场可由代码或资产名称自动匹配，例如 A 股、港股、美股、国内基金或现金/手动；当前价格/最新净值默认由系统主动获取，获取失败或尚未接入时允许用户手动补录，缺失时暂按成本价估值并在持仓台账中标记待获取价格；代码、开仓日期、期初价、汇率、现金流、费用、数据来源和复盘假设允许稍后补全，并在持仓台账中显示数据状态。
- 当前持仓和历史持仓列表。
- 账户总览、资产类型分布、市场分布、分布详情和总资产/收益率变化曲线。
- 单资产最近三个月历史价格或净值曲线。
- 资产变化归因，至少拆分净投入/提现、价格变动、汇率变动、分红/利息、手续费、税费、手动调整和未归因差异。
- 投资复盘笔记，支持草稿、私有和发布状态；公开内容以内嵌观点广场承载，不默认公开任何资产、持仓或交易信息。
- CSV / JSON 导入导出，本地或云端备份。
- 隐私说明、免责声明、风险提示和数据来源说明。

## 首版行情和净值范围

- A 股：沪深 300 成分股。
- 美股：纳斯达克 100 成分股。
- 港股：恒生科技指数成分股。
- ETF：沪深交易所上市 ETF、美股 ETF 和港股科技相关 ETF 白名单。
- 基金：国内公募开放式基金，按用户搜索或手动录入后拉取净值。
- 数据频率：日频收盘价或基金单位净值。
- 默认历史范围：最近三个月。

## 暂不做

- 当前 MVP 不提供完整云端资产同步和多设备一致性保证。
- 自动交易。
- 自动连接券商 API。
- 实时行情、盘口和分钟级行情。
- 全市场行情覆盖。
- 自动生成个性化投资建议。
- 收益承诺、资产排名、跟单暗示。
- 复杂组织权限和实时社区。

## 核心流程

### 注册登录

1. 用户使用邮箱验证码或 OAuth 登录。
2. 服务端创建用户和默认设置。
3. 所有私有数据按 `user_id` 隔离。
4. 登录、登出、导出、删除账户等敏感操作写入审计日志。

### 资产录入

1. 用户选择账户、资产类型、市场、代码、币种和交易类型。
2. 用户录入数量、价格、费用、税费、交易时间和数据来源。
3. 客户端做格式校验，服务端做权限和业务校验。
4. 写入交易流水，重新计算或更新持仓。
5. 如果行情或净值缺失，允许保存资产并标记数据缺失。

### 持仓和历史持仓

1. 当前持仓按账户和资产聚合。
2. 当前持仓支持分批卖出/减仓，扣减剩余数量并累计已实现收益。
3. 清仓后保留历史持仓、 realized PnL、清仓原因和审计记录，并以弹窗建议用户补充清仓复盘。
4. 当前持仓和历史持仓都必须能回溯到交易流水。

### 价格 / 净值更新

1. 首次上线回补首版白名单最近三个月数据。
2. 用户新增资产时异步创建回补任务。
3. 每日收盘后按市场拆分任务更新。
4. 基金净值按披露窗口更新，并在次日补跑处理迟到数据。
5. 展示层标注数据来源、价格日期、更新时间和缺失状态。

### 归因分析

1. 用户选择起始日期和结束日期。
2. 系统读取起止估值、交易流水、价格快照、汇率快照和费用税费。
3. 输出贡献项，并保证贡献项加总后等于组合价值变化。
4. 无法归因部分显式展示为未归因差异。

### 复盘笔记

1. 用户创建笔记，关联资产、账户、交易、时间区间和标签。
2. 默认状态为草稿或私有。
3. 发布内容必须避免收益承诺、跟单暗示和误导性表达。

### 导入导出

1. 用户上传或粘贴 CSV / JSON。
2. 系统先预览、校验、识别重复记录和字段错误。
3. 用户确认后写入，并保留导入批次和审计日志。
4. 导出文件不得包含服务端密钥或其他用户数据。

## 成功指标

- 用户能在 10 分钟内完成账户、资产和一条复盘笔记。
- 金融计算单元测试覆盖金额、数量、汇率、费用、税费、分红和归因对齐。
- 任一持仓市值可由交易、价格和汇率复算。
- 价格或净值缺失时页面显示明确状态，不静默使用错误数据。
- 导入失败能指出具体字段和原因。

## 合规文案基线

所有涉及收益、价格、社区观点和数据展示的位置都应包含或继承以下基线：

> 数据仅供记录和分析，不构成投资建议。价格和净值可能延迟或缺失，请以官方披露和交易确认记录为准。
