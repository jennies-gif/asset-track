# 产品定位

- 状态：已由产品负责人确认
- 适用阶段：个人产品 MVP 构建与早期验证
- 最后更新时间：2026-07-11
- 维护责任：重大定位变化必须经产品负责人确认

## 产品概述

资产轨迹是一款面向拥有多个投资账户，或同时持有不同市场、币种资产的中文个人投资者的资产追踪工具。

产品帮助用户统一记录和查看分散的投资资产，获得清晰、可信的整体组合视图；在此基础上进一步解释资产变化来源，并支持用户回看自己的投资判断。

## 产品价值层级

### 产品品类

个人多账户资产追踪工具。

### 当前 MVP 第一价值

帮助用户通过可接受的录入或导入成本，获得清晰、可信的整体投资资产视图。

用户第一次使用后，应该能够理解：

- 当前整体投资资产大约是多少；
- 资产主要分布在哪些账户、市场、币种和类别；
- 主要持仓和资产集中情况；
- 哪些数据完整可信；
- 哪些数据缺失、过期、手动填写或使用估算。

### 核心差异

产品不只展示资产结果，还解释资产变化可能来自：

- 净投入；
- 价格变化；
- 汇率变化；
- 分红；
- 费用与税费；
- 其他未归因差异。

所有关键价格、汇率和估值结果应尽量展示来源、时间、缺失和估算状态。

### 长期价值

将资产变化、交易行为和用户当时的投资判断关联起来，帮助用户持续回看和检验自己的投资过程。

## 目标用户

### 首批目标用户

拥有多个投资账户，或同时持有不同市场、币种资产，希望更清楚地查看整体资产状态的中文个人投资者。

### 必要特征

- 存在资产分散问题；
- 需要跨账户、市场、币种或资产类别查看整体资产；
- 愿意进行一定程度的数据录入、导入、校对和后续维护。

### 加分特征

- 有长期投资习惯；
- 会定期回顾资产；
- 关注数据来源、计算口径和结果可信度；
- 希望理解资产变化原因。

### 当前不要求

用户不需要已经具备成熟的投资复盘习惯。

### 当前非目标用户

- 主要追求实时行情、盘口和短线交易信号的用户；
- 希望获得个性化买卖建议、自动投顾或跟单服务的用户；
- 完全不愿意进行任何数据录入或校对，并要求全自动资产聚合的用户；
- 当前阶段需要复杂组织权限、审批和多人协作的团队用户。

## 核心用户问题

用户的资产分散在多个账户、市场、币种和工具中，难以形成清晰、可信的整体视图。

更进一步，用户看到资产增加或减少时，通常难以区分变化究竟来自：

- 资金投入或取出；
- 市场价格；
- 汇率；
- 分红；
- 费用和税费；
- 其他数据差异。

用户的交易原因、当时判断和后续结果也常常分散在不同工具中，难以长期回看和验证。

## 核心使用闭环

用户因资产分散或难以看清整体状态进入产品
→ 通过快速录入或导入建立账户、资产和持仓信息
→ 查看整体资产、分布、主要持仓和数据完整状态
→ 获得清晰、可信的组合全景
→ 随着交易、价格和汇率变化，进一步理解资产为什么发生变化
→ 记录或关联当时的投资判断
→ 在后续交易、数据更新或阶段回顾时再次回来维护和复盘。

## 首次体验成功标准

用户第一次使用后应当能够：

- 独立建立一个初步资产组合；
- 正确理解整体资产状态；
- 看清主要账户、资产类别、市场或币种分布；
- 知道哪些数字可信，哪些数据仍需补充；
- 明白后续为什么需要继续更新或维护数据。

首次使用不强制要求用户完成变化归因或投资复盘。

## 当前 MVP 核心假设

拥有多个账户、市场或币种资产的个人投资者，愿意付出可接受的数据录入与维护成本，换取一个比现有方式更清晰、可信、可持续更新的整体资产视图。

## 次级差异化假设

当用户已经建立可信组合视图后，变化归因能够显著增强产品价值，并使产品明显优于普通资产汇总工具。

## 长期留存假设

将资产变化、交易行为和投资判断关联起来，能够形成持续复盘和长期使用动力。

以上假设均处于待验证状态，不得写成已经验证的事实。

## 产品差异化

### 相比普通记账工具

重点不是日常收入与支出，而是投资资产、持仓、交易、估值、收益来源和投资过程。

### 相比单一券商 App

提供跨账户、跨市场和跨币种的整体资产视图，不以促成交易、实时行情或提高交易频率为目标。

### 相比普通投资组合工具

强化：

- 数据来源与更新时间；
- 缺失和估算状态；
- 变化来源解释；
- 未归因差异；
- 可复算和可下钻；
- 资产变化与投资复盘之间的关联。

## 数据存储与控制原则

### 长期原则

数据控制权与存储透明是长期原则。

用户应当能够清楚知道：

- 哪些数据保存在本地；
- 哪些数据会发送到服务器；
- 数据用于什么目的；
- 如何导出、备份、迁移和删除私人资产数据。

### 当前 MVP

当前 MVP 中：

- 用户账户、持仓、交易、现金流和复盘等私人资产数据以本地保存为主；
- 公共行情、基金净值、汇率和证券基础信息可以通过联网服务获取和缓存；
- 公共行情云端获取不等于用户私人资产数据已经云同步；
- 当前不提供完整的用户资产云同步和多设备一致性保证。

当前实际数据流仍应以代码和数据流审计结果为准，不得在未经技术确认的情况下对外承诺所有私人数据绝不离开设备。

### 长期方向

云端资产同步与多设备支持属于长期重要能力，包括：

- 用户账户；
- 云端安全存储；
- 自动备份；
- 数据恢复；
- 多设备同步；
- 同步冲突处理；
- 数据导出和云端删除。

该能力不属于当前 MVP 上线门槛，启动时间应结合真实用户对多设备、自动备份、恢复和持续使用的需求决定。

## 行情与公共数据边界

当前 MVP 为有限支持范围内的证券、基金和汇率提供日频或延迟行情自动更新，并保留手动价格和估值兜底。

产品不承诺：

- 实时行情；
- 全市场覆盖；
- 外部数据永不中断；
- 所有历史价格完整；
- 所有公司行为自动处理。

所有价格、净值和汇率应尽量展示：

- 数据来源；
- 更新时间；
- 延迟状态；
- 缺失状态；
- 手动填写或估算标识。

自动行情的目标是降低用户维护公共数据的成本，而不是将产品发展为行情终端。

## 复盘定位

复盘是产品的长期核心价值之一。

当前 MVP 保留轻量、私有的投资复盘能力，用于：

- 记录某项资产或交易的判断；
- 关联资产或交易；
- 保存时间和基础内容；
- 在未来重新查看；
- 从资产或分析结果进入复盘。

复盘不作为首次使用的强制步骤。

当前暂不建设：

- 复杂复盘模板；
- AI 自动生成投资结论；
- 自动判断投资决策对错；
- 复盘评分；
- 强制复盘流程。

## 社区定位

社区不属于当前 MVP。

只有在私有复盘价值和持续使用行为得到验证后，才重新评估是否支持：

- 选择性分享；
- 匿名化复盘案例；
- 决策过程分享；
- 复盘模板或案例库。

当前明确不做：

- 默认公开资产和持仓；
- 收益排行；
- 跟单；
- 达人体系；
- 实时社区；
- 投资观点推荐流；
- 资产数据与公开内容默认关联。

## 当前非目标

当前 MVP 不提供：

- 自动交易；
- 自动投顾；
- 个性化买卖建议；
- 收益承诺、保本暗示和跟单；
- 券商、银行和交易所自动连接；
- 实时行情、盘口和分钟级行情；
- 全市场行情覆盖；
- 完整公司行为自动处理；
- 复杂团队权限；
- 完整云端资产同步；
- 订阅支付；
- 完整社区和公开收益排行；
- 复杂估值模型、技术指标和短线交易信号。

## 已确认事实

- 当前产品是 Web 资产管理 MVP。
- 当前核心工作区包括总览、资产、分析和复盘。
- 当前已具备资产录入、持仓追踪、变化拆解、复盘和本地备份等能力。
- 当前行情、价格和趋势数据可能存在延迟、缺失或估算。
- 金融计算应当可解释、可复算和可测试。
- 用户资产、交易、身份和笔记属于敏感数据，默认不得公开。
- 产品仅用于记录和分析，不构成投资建议。

## 待验证假设

- 用户愿意以合理的数据维护成本换取整体资产视图；
- 用户认为整体资产视图明显优于现有使用方式；
- 用户能够理解数据来源、缺失和估算状态；
- 变化归因能够显著提高产品价值；
- 归因与复盘能够形成长期使用动力；
- 用户是否将本地保存和数据控制视为关键选择因素；
- 用户是否存在明显的云同步、多设备和自动备份需求；
- 有限日频行情覆盖是否足够支持首批目标用户；
- 首批目标用户是否愿意为相关能力付费。

## 领域风险与边界

- 所有展示仅用于资产记录和分析，不构成投资建议；
- 不得承诺收益、暗示保本或诱导跟单；
- 不得根据用户资产自动输出个性化交易建议；
- 价格、净值和汇率应注明来源、时间、延迟和缺失状态；
- 估算趋势不得被表达为真实历史净值；
- 用户资产、交易、身份和笔记按敏感数据处理；
- 未经用户明确授权，不得默认公开或分享私人资产数据；
- 外部行情源的授权、稳定性、覆盖率和商业成本仍需持续评估；
- 数据存储和传输边界应以实际代码与数据流审计结果为准。
