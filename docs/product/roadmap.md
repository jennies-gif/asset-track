## Later

### 前端单入口收敛与 React 迁移评估

状态：长期待办，尚未决定迁移，启动前需由产品负责人确认。

当前正式线上入口是根目录原生 JavaScript 静态 MVP（`index.html`、`src/`）；`apps/web/` 是尚未达到功能等价的 React + TypeScript 并行入口。种子试用版本冻结和核心价值验证完成前，不在 React 入口重复开发正式产品功能，只维护必要的类型、构建、安全和共享领域契约。

满足以下条件后，单独评估是否由 React 入口接替正式产品：

- 核心产品价值经过真实种子用户验证，产品值得进入持续投入阶段；
- 原生 JavaScript 入口已出现明确、可举证的维护阻碍，而不是仅因为技术栈不够新；
- 已建立覆盖总览、资产、交易、分析、行情、复盘、导入导出和设置的功能等价清单；
- 已确认浏览器本地数据、导入导出格式和 API 契约向后兼容；
- 已具备自动测试、桌面与移动端验收、部署切换和回滚方案；
- 迁移周期内能够控制旧入口新增功能，避免两套产品继续扩大差距。

如决定迁移，完成标准不是“React 页面可以运行”，而是 React 入口通过完整功能等价、数据兼容和核心路径验收，正式部署切换后停止维护旧入口。未经上述评估和确认，不切换 Vercel 正式入口，不删除当前可用版本，也不启动大规模重写。

### 云端资产同步与多设备支持

在核心价值和持续使用需求得到验证后，建设用户账户、加密云端存储、自动备份、多设备同步和数据恢复能力。启动前需验证用户需求、隐私接受度及双模式维护成本。

## 明确暂不做

- MVP 不提供实时行情、全市场覆盖以及完整公司行为自动处理。

## 重新评估条件

- 种子试用版本完成冻结并获得真实用户验证结果后，重新评估前端单入口收敛与 React 迁移。
