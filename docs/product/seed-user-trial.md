# 种子用户试用说明

本文件用于邀请第一批种子用户试用资产轨迹 MVP，并引导他们完成核心路径后提交有效反馈。

## 试用定位

资产轨迹当前是本地优先 MVP，用于记录资产、查看持仓和收益变化、理解资产变化来源，并沉淀投资复盘。

当前版本不提供投资建议、收益承诺、保本暗示、自动交易、券商同步或云端多设备同步。数据默认保存在用户当前浏览器本地，集中录入或重要修改后需要导出 JSON 备份。

## 邀请文案

### 简短版

我在做一个本地优先的资产记录和投资复盘工具，想邀请你试用 10 分钟。

它主要帮你做三件事：

- 记录多账户、多资产、多币种持仓。
- 看清当前资产、市值、收益和变化来源。
- 把买入、卖出或清仓后的判断沉淀成复盘。

当前版本是 MVP，不连接券商，不上传资产数据，也不提供投资建议。你可以用示例数据试，也可以录入少量虚构或脱敏资产体验流程。试完后请点页面底部“用户反馈”，告诉我哪里卡住、哪里不可信、哪里值得继续做。

### 完整版

我在做一个叫“资产轨迹”的资产记录和投资复盘工具，想请你作为种子用户试用。

这个版本的重点不是行情推荐或交易建议，而是帮助个人投资者把资产事实记录清楚：

- 当前持有什么资产，在哪个账户，成本和估值是多少。
- 总资产变化来自净投入、价格、汇率、分红、费用还是未归因差异。
- 每次买入、卖出或清仓后，能不能留下复盘，之后回看当时的判断。

当前是本地优先 MVP：

- 数据保存在你的浏览器本地，不会主动上传。
- 不连接券商、银行或交易所。
- 不承诺实时行情或全市场覆盖。
- 所有收益、风险和图表仅用于记录与复盘，不构成投资建议。
- 请不要录入真实账户凭据、身份证明或完整敏感资产明细。

建议你用 10 分钟按下面路径试一次，然后通过页面底部“用户反馈”把感受发给我。

## 10 分钟试用路径

1. 打开总览页，先看整体信息密度和数据提示是否能理解。
2. 到资产页添加 2-3 个资产，可以使用虚构或脱敏数据。
3. 至少添加一种非现金资产和一种现金或低风险资产。
4. 查看当前持仓、资产配置和数据状态。
5. 对一个已有持仓记录一次加仓、减仓或清仓。
6. 到分析页查看总资产趋势、归因拆解、收益表现和数据口径标签。
7. 到复盘页写一条简短复盘，并关联资产或交易。
8. 导出 JSON 备份，确认你理解本地数据和备份边界。
9. 点击页脚“用户反馈”，提交试用反馈。

## 重点观察问题

请种子用户尽量回答以下问题：

- 你是否能在 10 分钟内完成添加资产、查看分析、写复盘和导出备份？
- 哪一步最卡？是字段看不懂、按钮找不到，还是不确定数据含义？
- 哪个数字或图表让你不信任？为什么？
- 图表中的“当前录入 / 估算趋势 / 行情缓存 / 演示估算”等口径是否清楚？
- 添加资产和记录交易是否分得清？
- 本地保存和 JSON 备份的边界是否让你安心，还是会阻碍继续使用？
- 哪个功能最有价值？
- 如果继续使用，你最希望下一步补什么？
- 你是否愿意为这个工具付费？愿意或不愿意的原因是什么？

## 反馈入口说明

页面底部有“用户反馈”按钮。点击后会生成结构化反馈模板，并提供复制或邮件草稿发送能力。

反馈模板只包含试用概况统计，不会自动附带资产名称、金额、备份文件或具体持仓明细。

反馈时请不要发送：

- 完整 JSON 备份文件。
- 真实券商、银行、交易所账户凭据。
- 身份证、护照、银行卡等身份或支付信息。
- 完整真实资产明细。

## 内部判定标准

本轮种子用户试用重点不是验证所有高级功能，而是验证三件事：

- 用户是否理解产品定位：本地优先资产记录 + 分析 + 复盘。
- 用户是否信任数据口径：哪些是真实录入，哪些是估算，哪些来自行情缓存。
- 用户是否愿意继续记录：资产、交易和复盘是否形成长期使用动机。

如果 3-5 名种子用户中至少 2 名能完成核心路径，并愿意继续使用或提出明确付费条件，则下一轮优先打磨历史价格、导入体验、备份安全和资产录入效率。
