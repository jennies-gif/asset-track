# 资产轨迹视觉规范

本文面向「资产轨迹」产品设计与前端实现。产品定位是长期主义、理性投资、资产记录、归因分析、投资复盘和风险控制；视觉系统应帮助用户看清事实、理解变化、保持冷静，不制造交易冲动。

## 0. 实现源头

视觉 token 的代码源头固定为：

```text
src/styles/app.css
```

其中 `:root` 顶部变量是全站样式的唯一实现标准。新增 UI 时应优先引用 token，不直接散写圆角、阴影、字重和字体族。

### 0.1 固定 Token

| Token | 当前值 | 用途 |
| --- | --- | --- |
| `--font-body` | 系统无衬线字体栈 | 全站正文、表单、按钮 |
| `--font-display` | `var(--font-body)` | 页面标题、区块标题 |
| `--font-number` | DIN / Roboto Mono / SFMono 等宽数字字体栈 | 金额、收益率、价格、数量 |
| `--radius-control` | `6px` | 按钮、输入框、菜单项、页签 |
| `--radius-card` | `6px` | 指标卡、列表项、状态块、表格容器 |
| `--radius-panel` | `6px` | 页面主要面板、弹窗、浮层 |
| `--radius-pill` | `999px` | 徽章、胶囊筛选、圆形状态点 |
| `--shadow-card` | `none` | 普通面板和卡片 |
| `--shadow-popover` | 中等阴影 | 下拉菜单、浮层 |
| `--shadow-dialog` | 较强阴影 | 模态框 |
| `--weight-normal` | `400` | 正文 |

- 页面新增样式不得再直接写 `border-radius: 18px`、`font-weight: 650/750/800/900` 这类临时值，除非有明确例外并在注释中说明。
- 大部分界面元素只使用 `400 / 500 / 600 / 700` 四档字重。
- 普通面板和卡片保持无阴影，使用 `1px solid #E5E7EB` 的浅灰实体描边；浮层和弹窗才使用对应阴影 token。
- 胶囊形和圆形元素使用 `--radius-pill`，不要散写 `999px`。

## 1. 设计原则

### 1.1 专业可信

- 首屏直接进入资产概览、持仓、分析或复盘，不做营销式大幅宣传页。
- 数据来源、价格时间、计算口径、缺失状态和风险提示应可见。
- 大额资产、收益率、回撤、归因结果等关键数字必须拥有稳定排版，不因内容长短导致布局跳动。

### 1.2 冷静理性

- 避免强刺激颜色、大面积渐变、夸张动效、榜单排名和煽动性文案。
- 正负收益可以被识别，但不要把涨跌做成“兴奋/恐慌”视觉。
- 分析页重点表达“发生了什么、为什么发生、哪里需要检查”，不表达“该买/该卖”。

### 1.3 克制高级

- 使用低饱和中性色、大面积留白、清晰分隔线和紧凑表格。
- 组件圆角控制在 `6-10px`：控件 `6px`，卡片 `8px`，主要面板 `10px`；阴影轻微，避免厚重卡片堆叠。
- 页面信息密度应偏工具型，适合长期反复查看。

### 1.4 有温度但不过度活泼

- 在复盘、提醒、空状态中可以使用更柔和的语言和浅色背景。
- 温度来自清晰解释、可追溯证据和减少焦虑，而不是插画堆叠或游戏化奖励。

### 1.5 可审计

- 所有指标附近应能下钻到交易、价格、汇率、费用或手动调整来源。
- “未归因差异”“价格过期”“来源缺失”要作为明确状态展示，不隐藏在角落。

## 2. 色彩系统

### 2.1 品牌基调

推荐品牌气质：`浅灰页面 + 白色内容面 + 青绿色主色 + 石墨文字 + 克制语义色`。参考 Ghostfolio 的清爽资产看板气质，但不照搬营销页表达：页面应更像长期使用的资产管理工具。

这套颜色适合资产管理产品的原因：

- 白色内容面和浅灰页面让表格、图表、筛选器边界更清楚，减少“脏”和“糊”的感觉。
- 青绿色主色比传统金融蓝更轻，适合作为品牌识别和主要行动色。
- 蓝色只用于信息、链接、可下钻入口，不和收益涨跌混用。
- 红绿只用于收益、亏损和校验状态，不把页面变成行情软件。

### 2.2 基础色 Token

| Token | 色值 | 用途 |
| --- | --- | --- |
| `--bg` | `#F8F9FA` | 页面背景，干净浅灰，承托工具界面 |
| `--surface` | `#FFFFFF` | 表格、面板、弹窗、输入框背景 |
| `--surface-muted` | `#F1F3F5` | 次级区域、筛选栏、浅提示背景 |
| `--ink` | `#212529` | 主文字，接近石墨黑 |
| `--muted` | `#6C757D` | 全站辅助文字、说明、表头、弱说明、禁用文本 |
| `--subtle` | `#6C757D` | 兼容旧样式的弱文字别名，新样式不要直接使用 |
| `--line` | `#E9ECEF` | 分隔线、输入框边框 |
| `--line-strong` | `#DEE2E6` | 表格冻结列、重要分区边界 |

### 2.3 主色与辅助色

| Token | 色值 | 使用场景 |
| --- | --- | --- |
| `--primary` | `#36CFCC` | 主按钮、当前导航、关键可点击状态 |
| `--primary-hover` | `#22B8CF` | 主按钮 hover/active |
| `--primary-soft` | `#E3FAFC` | 当前筛选、选中标签、低强度品牌背景 |
| `--accent` | `#3686CF` | 链接、信息状态、可下钻入口、图表强调线 |
| `--accent-soft` | `#E7F5FF` | 信息提示、数据来源说明 |
| `--warning` | `#F59F00` | 复盘提醒、待确认、需要人工检查 |
| `--warning-soft` | `#FFF9DB` | 温和提醒背景 |

使用建议：

- 主色只承担产品识别和主要行动，不用于大面积背景。
- 蓝色辅助色用于“可查看、可下钻、信息解释”，不要和收益涨跌混用。
- 暖色用于“需要停下来确认”，不要直接等同于亏损。

### 2.4 图表色

图表色应独立于页面装饰色管理，避免环形图、趋势线、瀑布图各用一套临时颜色。

| Token | 色值 | 使用场景 |
| --- | --- | --- |
| `--chart-1` | `#36CFCC` | 第一序列、总计、核心资产分类 |
| `--chart-2` | `#3686CF` | 第二序列、当前配置、信息型图表 |
| `--chart-3` | `#12B886` | 正贡献、收入、正向柱状 |
| `--chart-4` | `#F59F00` | 回撤、警戒线、待检查项 |
| `--chart-5` | `#7C3AED` | 第五序列，少量辅助分类 |
| `--chart-6` | `#6C757D` | 其他、未分类、低优先级序列 |
| `--chart-line` | `#22B8CF` | 主趋势折线 |
| `--chart-grid` | `#E9ECEF` | 图表网格线 |
| `--chart-axis` | `#DEE2E6` | 坐标轴 |
| `--chart-muted` | `#6C757D` | 标签、目标配置、辅助点线 |

规则：

- 分类图只使用 `--chart-1` 到 `--chart-6`，不要混入黑色、纯红、纯绿。
- 趋势图主线使用 `--chart-line`，网格线和轴线保持低对比。
- 正负贡献仍使用语义色，但面积颜色要克制，避免行情软件感。
- 页面上的灰色字体统一使用 `--muted`；`--subtle` 只作为兼容别名保留，不再用于新增字体样式。

### 2.5 涨跌与状态色

面向中文投资者，默认采用 `红涨绿跌`，同时用符号、文本和图形方向辅助识别，避免只依赖颜色。

| 语义 | Token | 色值 | 使用方式 |
| --- | --- | --- | --- |
| 盈利 / 正贡献 | `--positive` | `#12B886` | 收益为正、贡献为正、校验通过 |
| 盈利浅底 | `--positive-soft` | `#E6FCF5` | 正贡献背景、图表浅面积 |
| 亏损 / 负贡献 | `--negative` | `#DC3545` | 收益为负、贡献为负、错误提示 |
| 亏损浅底 | `--negative-soft` | `#FFF5F5` | 负贡献背景、图表浅面积 |
| 风险 | `--risk` | `#E8590C` | 高集中度、最大回撤超阈值、风险灯 |
| 风险浅底 | `--risk-soft` | `#FFF4E6` | 风险提示背景 |
| 成功 / 已完成 | `--success` | `#12B886` | 导入成功、备份完成、校验通过 |
| 警告 / 待处理 | `--warning` | `#F59F00` | 价格过期、汇率缺失、待复盘 |
| 错误 / 阻断 | `--danger` | `#DC3545` | 表单错误、导入失败、权限拒绝 |
| 信息 | `--accent` | `#3686CF` | 数据说明、来源、帮助入口 |

规则：

- 收益数字前保留 `+` / `-`，例如 `+3.42%`、`-1.08%`。
- 表格中正负值不加大面积色块，只改变文字色和轻量箭头。
- 风险色优先用于“风险程度”，不要用红色表达所有负面状态，否则会和上涨混淆。
- 海外用户或未来多地区版本可在设置中提供 `红涨绿跌 / 绿涨红跌` 切换。

## 3. 字体系统

### 3.1 字体家族

推荐 CSS：

```css
--font-sans: Inter, "Avenir Next", ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif;
--font-number: "DIN Alternate", "Roboto Mono", "SFMono-Regular", ui-monospace, Consolas, monospace;
--font-serif: "Noto Serif SC", "Songti SC", Georgia, serif;
```

使用建议：

- 中文界面默认使用系统无衬线，保证清晰和兼容。
- 资产金额、收益率、数量、价格、日期可使用数字字体或开启 `font-variant-numeric: tabular-nums;`，保证纵向对齐。
- 复盘正文可继续使用无衬线；长文阅读模式可选衬线，但不要作为默认全站字体。

### 3.2 字号层级

| 层级 | 字号 / 行高 | 字重 | 场景 |
| --- | --- | --- | --- |
| 页面标题 | `24px / 32px` | `700` | 总览、资产、分析、复盘一级标题 |
| 区块标题 | `18px / 26px` | `700` | 图表区、表格区、表单分组 |
| 小标题 | `15px / 22px` | `650` | 卡片标题、弹窗标题、设置项 |
| 正文 | `14px / 22px` | `400` | 表单、说明、正文内容 |
| 表格正文 | `13px / 20px` | `400` | 持仓、交易、归因表格 |
| 辅助说明 | `12px / 18px` | `400-500` | 来源、时间戳、脚注、风险提示 |
| 标签 | `12px / 16px` | `600` | 状态、分类、筛选条件 |
| 关键金额 | `28-32px / 36px` | `700` | 总资产、区间收益 |
| 次级数据 | `20-24px / 30px` | `700` | 未实现收益、最大回撤、现金流 |
| 表格数字 | `13px / 20px` | `500-650` | 市值、数量、成本、收益率 |

规则：

- 中文字体不要使用负字距。
- 数据数字统一右对齐，列宽稳定。
- 关键金额可以大，但同屏最多 1-2 个超大数字，避免主次混乱。
- 辅助说明不能低于 `12px`，合规和数据来源信息要可读。

## 4. 组件规范

### 4.1 页面布局

- 桌面端推荐最大宽度 `1240-1280px`，页面左右边距 `24-32px`。
- 主导航固定为：总览、资产、分析、复盘。
- 页面用全宽区块和表格组织信息，不把每个区域都做成大卡片。
- 复杂页面采用 `顶部筛选栏 + 核心指标 + 主图表/主表格 + 明细抽屉`。

### 4.2 面板与卡片

- 面板用于承载图表、表格、表单；圆角 `8px`，边框 `1px solid --line`。
- 卡片只用于少量关键指标、重复列表项、弹窗内容，不做卡片套卡片。
- 指标卡结构：标题、主数值、变化值、来源/时间戳。缺一不可，尤其是来源。
- 面板阴影使用极轻阴影或不用阴影，主要靠边框和间距区分。

### 4.3 按钮

| 类型 | 视觉 | 场景 |
| --- | --- | --- |
| 主按钮 | 深墨绿底、白字 | 添加资产、保存、确认导入 |
| 次按钮 | 白底、边框、深色字 | 取消、导出、恢复示例 |
| 文本按钮 | 无底色、主色文字 | 查看来源、下钻明细 |
| 危险按钮 | 白底红字或红色边框 | 删除、清空、撤销导入 |

规则：

- 按钮高度桌面 `36-40px`，移动端 `40-44px`。
- 同一区块主按钮最多一个。
- “删除资产”“清空数据”等危险操作必须二次确认，并说明后果。
- 图标按钮必须有 tooltip 或 aria label。

### 4.4 标签与状态

标签分三类：

- 分类标签：资产类别、市场、账户类型，使用中性底色。
- 状态标签：价格缺失、待复盘、已发布、私有，使用语义色。
- 筛选标签：当前筛选条件，使用主色浅底。

样式建议：

- 高度 `24-28px`，圆角 `6px` 或胶囊形。
- 文案短，不超过 6 个中文字符；长文放 tooltip 或详情。
- 状态标签必须含文本，例如“价格过期”，不要只放色点。

### 4.5 表单

- 资产录入默认快速模式，必要字段清晰分组：资产、账户、数量价格、费用税费、时间来源。
- 输入框高度 `38-42px`，错误信息紧贴字段下方。
- 金额、数量、价格、费率、汇率输入应明确单位和精度。
- 表单错误使用“字段 + 原因 + 修正方式”，例如“数量必须大于 0，请输入持有份额”。

### 4.6 表格

- 资产、交易、归因、导入预览优先使用表格。
- 第一列左对齐，数字列右对齐，状态列居中或左对齐。
- 表头使用 `12px` 辅助色，正文 `13px`。
- 支持排序、筛选、固定首列、行展开和横向滚动。
- 表格 hover 只用浅背景，不改变列宽或字体大小。

## 5. 图表规范

### 5.1 总体原则

- 图表用于解释组合价值、收益变化、归因拆解、资产分布和时间趋势。
- 不添加无决策价值的装饰图表。
- 图表必须配套数据口径、时间范围、更新时间和空/缺失状态。
- 可点击图表元素应能下钻到明细表或来源。

### 5.2 图表类型

| 场景 | 推荐图表 | 避免 |
| --- | --- | --- |
| 总资产趋势 | 折线图 / 面积很浅的折线图 | 3D 图、过粗面积图 |
| 收益率趋势 | 折线图，零轴明确 | 双轴复杂图 |
| 归因拆解 | 瀑布图 / 横向条形图 | 饼图解释变化原因 |
| 贡献排行 | 正负横向条形图 | 只用颜色不标数值 |
| 资产分布 | 条形占比 / Treemap / 简洁环图 | 多层玫瑰图 |
| 最大回撤 | 折线 + 回撤阴影 / 回撤条形 | 红色大面积警报 |
| 数据质量 | 状态列表 / 小型进度条 | 仪表盘式夸张评分 |

### 5.3 图表色板

分类色建议低饱和并保持足够区分：

| 类别 | 色值 |
| --- | --- |
| 股票 | `#2D6CDF` |
| 基金 / ETF | `#245B4E` |
| 现金 | `#66736D` |
| 债券 | `#8B6F47` |
| 数字资产 | `#7C5CC4` |
| 房产 / 其他 | `#B7791F` |
| 未分类 / 缺失 | `#A7B0AA` |

规则：

- 同一资产类别在全站颜色一致。
- 正负贡献使用涨跌色，不使用分类色。
- 缺失、未归因、待确认用灰或暖色，不用红绿。

## 6. 数据展示风格

### 6.1 总览

首屏优先级：

1. 总资产：最大字号，显示计价货币。
2. 区间收益与收益率：紧邻总资产，显示时间范围。
3. 数据状态摘要：价格缺失、汇率缺失、手动估值数量。
4. 总资产趋势：默认近一年，可切换区间。
5. 资产类别、市场、账户分布：用于回答“钱在哪里”。

展示规则：

- 总资产下方显示“截至日期 / 价格来源 / 汇率时间”。
- 收益率必须说明区间，例如“近一年收益率”。
- 缺失价格时显示估值策略，例如“暂按成本价估值”。

### 6.2 资产与持仓

持仓表推荐列：

- 资产名称 / 代码 / 市场
- 账户
- 数量
- 成本价 / 现价
- 市值
- 未实现收益
- 已实现收益
- 数据状态
- 操作

规则：

- 资产名称左侧可放小分类色条，不使用大图标堆叠。
- 市值、收益、收益率右对齐。
- 价格日期和来源放在现价下方小字。
- 数据状态必须能直接进入补录或查看来源。

### 6.3 分析

分析页应围绕三个问题组织：

1. 这段时间为什么变了。
2. 哪些资产贡献最大或拖累最大。
3. 哪些风险和数据问题需要检查。

推荐信息结构：

- 顶部：日期区间、账户/类别/市场筛选、计价货币。
- 核心：组合价值变化、净投入、价格变动、汇率变动、分红利息、费用税费、未归因差异。
- 中部：归因瀑布图 + 贡献排行表。
- 侧边或下方：风险灯和数据质量清单。
- 底部：可关联复盘的事件列表。

未归因差异：

- 始终显示。
- 若为 0，可显示“已对齐”。
- 若非 0，使用警告色并提供“查看缺失价格/汇率/交易”入口。

### 6.4 复盘

复盘页视觉应更安静，鼓励用户写下事实和判断：

- 列表侧重标题、关联资产、时间区间、状态、最后编辑时间。
- 编辑器使用清晰段落和小标题，不做强社交化动态流。
- 公开复盘必须弱化收益数字，强化风险提示和个人观点边界。
- 草稿、私有、发布状态用标签明确区分，默认私有或草稿。

### 6.5 社区 / 观点广场

- 不展示用户个人资产、持仓和交易细节，除非用户主动选择并经过确认。
- 不做收益排行、跟单按钮、达人收益榜。
- 帖子卡片显示观点标题、风险标签、发布时间、审核状态和举报入口。
- 风险提示常驻或继承在社区模块顶部。

## 7. 情绪与风险提示

### 7.1 冷静提醒

适用场景：

- 单日大涨大跌。
- 用户频繁修改同一资产。
- 清仓、重仓、集中度过高。
- 复盘长时间未完成。

视觉语言：

- 使用暖色浅底、左侧细色条、短文案。
- 不使用弹窗轰炸；重要阻断才用确认弹窗。
- 文案表达“暂停确认事实”，不指挥用户交易。

示例文案：

- “这笔调整会显著改变组合集中度，建议确认交易记录和原因后再保存。”
- “当前价格数据已超过 7 天未更新，分析结果可能偏离实际。”
- “该资产贡献异常，建议关联一条复盘记录，方便日后回看当时判断。”

### 7.2 风险灯

风险灯分级：

| 等级 | 视觉 | 含义 |
| --- | --- | --- |
| 正常 | 绿色文字或中性标签 | 数据完整、风险在阈值内 |
| 关注 | 暖色标签 | 需要查看，但不阻断 |
| 高风险 | 橙棕标签 + 说明 | 集中度、回撤、缺失数据明显 |
| 阻断 | 红色错误 | 导入失败、计算无法对齐、权限问题 |

风险灯必须说明触发原因，例如“单一资产占比 42%，超过 30% 提醒线”。

## 8. 页面应用建议

### 8.1 总览页

- 顶部四个指标以内：总资产、区间收益、区间收益率、数据状态。
- 图表和分布区保持简洁，不重复展示复杂归因。
- 分布详情用表格或紧凑列表，不用大面积卡片。

### 8.2 资产页

- 资产录入用右侧抽屉或页面内表单，保存后回到台账。
- 当前持仓与历史持仓使用 tab。
- 批量导入必须先展示预览、错误和重复项，再允许确认。

### 8.3 分析页

- 首屏不做资产分布重复图。
- 重点展示归因、贡献排行、风险灯、数据质量。
- 所有异常项都提供下一步入口：补价格、补汇率、查看交易、写复盘。

### 8.4 复盘页

- 采用笔记工具式布局：列表 + 编辑区 + 关联信息。
- 正文区域留白可以更舒展，但关联资产和数据证据保持结构化。
- 公开复盘入口放次级位置，避免产品变成社区优先。

### 8.5 移动端

- 总览优先显示总资产、区间收益、数据状态和趋势。
- 资产页支持快速录入和搜索。
- 复杂归因表格允许横向滚动，并提供摘要卡。
- 复盘阅读和编辑要优先保证输入体验。

## 9. 竞品参考与差异化

### 9.1 可借鉴

- Ghostfolio：组合追踪、资产配置、基准、费用分红的长期视角。
- Beavern：实时分析、阶段对比、资产筛选的清晰边界。
- Wealthfolio：本地优先、隐私友好、工具型桌面气质。
- Beancount / Fava：账本可审计、可查询、可复算的信任基础。
- Portfolio Performance：现金流、费用、税费、收益表现和归因的专业表达。
- Notion 类工具：复盘笔记的编辑体验、标签、关联和轻量数据库结构。

### 9.2 应避免

- 证券交易 App 的强行情刺激、盘口氛围、涨跌榜和高频交易暗示。
- 记账 App 过度生活化、卡通化、低信息密度的资产表达。
- 社区 App 的关注流、热榜、收益晒图和跟单暗示。
- Notion 风格的过度灰白与弱层级，金融关键数据必须更明确。
- 大面积深蓝、紫色渐变或科技感装饰，容易显得模板化且不够克制。

### 9.3 差异化方向

资产轨迹的视觉差异化应来自：

- “数据可信度”可见：每个关键数字都有来源、时间和缺失状态。
- “变化原因”可见：归因图和贡献表比行情涨跌更重要。
- “复盘证据链”可见：资产、交易、价格和笔记互相关联。
- “冷静辅助”可见：风险提示像审计提醒，不像营销刺激。

## 10. 禁忌事项

- 不使用“稳赚”“跑赢必备”“智能荐股”“买入信号”等投顾化表达。
- 不把红色用于所有负面状态，以免和中文语境中的上涨混淆。
- 不隐藏价格过期、汇率缺失、未归因差异和手动估值。
- 不使用真实用户资产、真实交易记录或真实身份信息作为示例数据。
- 不用大面积卡片堆叠稀释表格和明细。
- 不做收益排行榜、跟单入口、高手榜、晒单刺激。
- 不在 UI 中承诺收益、暗示保本或引导高风险投资。
- 不用装饰性图表替代可复算的指标说明。
- 不让表单只在客户端校验，服务端错误也必须有清晰展示。
