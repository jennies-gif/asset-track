# 周度复盘

## 周度复盘目的

用于定期对齐阶段目标、实际进展、产品质量、用户验证、风险和下一周期取舍。

## 触发时机

按固定周度节奏触发；当阶段目标、主要风险或上线判断发生重大变化时，也可提前触发。

## 会前输入

- 当前阶段与本周期目标。
- 上周期任务、交付物和验证结果。
- 用户反馈、产品数据和异常现象。
- 设计、技术、数据、安全与合规风险。
- 待产品负责人确认的决策点。

## 固定议程

1. 回看上周期唯一目标和完成情况。
2. 评审产品流程、UI、技术健康度、用户验证和风险。
3. 明确当前主要矛盾。
4. 确认下周期唯一目标、关键任务和明确不做事项。
5. 确认所需 Skill、负责人和下次复盘触发条件。

## 单次文件命名

单次周会记录使用 `YYYY-Www.md`，例如 `2026-W28.md`。

## 维护规则

- 一次复盘对应一个文件，不覆盖历史记录。
- 事实、判断、假设和决定应明确区分。
- 重大决定另行写入 Decision Log，并从周会记录链接。
- 未经产品负责人确认的产品结论不得标记为最终决定。
